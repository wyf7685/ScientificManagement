# 关于动态字段配置的说明

本文档主要用于说明系统中成果模型（Achievement）的动态扩展字段配置情况。
针对不同成果/项目类型（`achievement_types`），我们会动态关联并渲染相关的扩展字段（`achievement_field_defs`）。下方列出了目前存在的各分类及其配置的动态扩展字段规则和对应的字段类型，以供参考与对接。

## 1. 静态主表通用字段
所有类型的成果均包含主表（`achievement_mains`）通用字段。当出现以下字段时，无需单独走动态字段配置：
- **标题类**：关联主表 `title`，页面表单表现为“论文标题”、“专利名称”、“论文名称”、“项目名称”等。
- **编号类**：关联主表 `project_code`，页面表单表现为“论文编号”、“编号”。
- **作者/人员**：关联主表 `authors` (JSON 数据)，页面表单表现为“作者列表”。

---

## 2. 动态扩展分类与配置字段明细

以下归纳了基于 `achievement_field_defs` 表动态解析挂载到独立成果类型（`achievement_types`）上的字段映射与具体类型：

### 【专利 (Patent)】配置字段：
- **申请号**：普通文本类型 (`TEXT`, `application_no`)
- **专利类型**：普通文本类型 (`TEXT`, `patent_type`)
- **申请日期**：日期类型 (`DATE`, `application_date`)
- **申请人/专利权人**：普通文本类型 (`TEXT`, `applicant`)
- **发明人列表**：普通文本类型 (`TEXT`, `inventors`)

### 【论文 (Paper)】配置字段：
- **第一作者**：普通文本类型 (`TEXT`, `first_author`)
- **发表刊物/会议**：普通文本类型 (`TEXT`, `pub_venue`)
- **发表日期**：日期类型 (`DATE`, `pub_date`)
- **期刊/会议级别**：普通文本类型 (`TEXT`, `journal_level`)
- **通讯作者**：普通文本类型 (`TEXT`, `corresponding_author`)
- **影响因子**：普通文本类型 (`TEXT`, `impact_factor`)
- **是否TOP**：布尔单选类型 (`BOOLEAN`, `is_top_journal`)

### 【软著 (Software)】配置字段：
- **登记号**：普通文本类型 (`TEXT`, `registration_no`)
- **版本号**：普通文本类型 (`TEXT`, `version_no`)
- **登记批准日期**：日期类型 (`DATE`, `approve_date`)
- **著作权人**：普通文本类型 (`TEXT`, `right_holder`)
- **开发完成日期**：日期类型 (`DATE`, `completion_date`)

### 【数据集 (Dataset)】配置字段：
- **数据集标识**：普通文本类型 (`TEXT`, `dataset_identifier`)
- **存储平台/仓库**：普通文本类型 (`TEXT`, `repository_name`)
- **发布日期**：日期类型 (`DATE`, `publish_date`)
- **数据格式**：普通文本类型 (`TEXT`, `data_format`)
- **样本量/记录数**：普通文本类型 (`TEXT`, `sample_count`)

### 【申报书 (Proposal File)】配置字段：
- **项目所属领域**：富文本类型 (`RICHTEXT`, `project_field`)
- **项目名称**：富文本类型 (`RICHTEXT`, `project_name`)
- **项目描述**：普通文本类型 (`TEXT`, `project_description`)
- **项目预期成果**：普通文本类型 (`TEXT`, `expected_achievement`)
- **研究周期（月）**：数字类型 (`NUMBER`, `research_period`)

### 【征集附件 (Collection Attachment)】配置字段：
- **征集主题名称**：富文本类型 (`RICHTEXT`, `collection_name`)
- **征集描述**：普通文本类型 (`TEXT`, `description`)
- **开始时间**：日期+时间类型 (`DATETIME`, `start_time`)
- **测试时间**：日期类型 (`DATE`, `time`)

### 【合同相关】配置字段：
- **合同模版 (Contract Template)**：
  - **合同备注**：富文本类型 (`RICHTEXT`, `contract_remark`)
- **已签署合同 (Signed Contract)**：
  - **合同备注**：普通文本类型 (`TEXT`, `contract_remark`)

### 【成果物报告 (Deliverable Report)】配置字段：
- **报告填写要求**：富文本类型 (`RICHTEXT`, `request`)

## 3. 已知冗余与测试字段说明
部分历史导入或开发时出现过、但未正式在模型中作为独立动态字段配置登记的内容：
- **测试字段**：仅为开发人员用于验证动态表单联动的临时字段，可忽略。
- **研究领域 / 论文范围**：数据库中并无独立存在的对应键值，业务上这两项已经被“项目所属领域（`project_field`）”或作为普通静态数据涵盖。
- **论文发布日期**：系业务口径融合称呼，实际上底层将其严谨剖分为论文类型的“**发表日期 (pub_date)**”与数据集类型的“**发布日期 (publish_date)**”。
