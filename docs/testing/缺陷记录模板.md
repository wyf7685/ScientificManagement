# ScienceManager 缺陷记录模板

## 1. 缺陷单模板

| 字段 | 内容要求 |
|---|---|
| 缺陷ID | 统一编号，如 `BUG-20260306-001` |
| 标题 | 模块 + 现象，简明明确 |
| 提交日期 | 发现日期 |
| 提交人 | 测试人员姓名 |
| 环境 | SIT/浏览器/接口版本/账号角色 |
| 模块 | 认证、成果管理、审核、过程系统等 |
| 严重级别 | `P0` / `P1` / `P2` / `P3` |
| 优先级 | `High` / `Medium` / `Low` |
| 前置条件 | 触发问题所需账号、数据、状态 |
| 复现步骤 | 编号列出，确保他人可复现 |
| 实际结果 | 实际页面表现、接口响应、报错内容 |
| 预期结果 | 正常应有行为 |
| 影响范围 | 单用户、全角色、核心主流程、数据安全等 |
| 附件 | 截图、录屏、请求报文、响应报文、日志片段 |
| 复现率 | 必现 / 高概率 / 低概率 |
| 根因分析 | 开发定位后补充 |
| 修复版本 | 计划修复版本号 |
| 修复人 | 开发负责人 |
| 状态 | 新建、已确认、修复中、待回归、已关闭、拒绝 |
| 回归结果 | 通过 / 失败 / 部分通过 |
| 备注 | 风险说明、绕过方案、关联缺陷 |

## 3. 填写规范

1. 标题必须体现模块、动作和异常现象，避免使用“有问题”“异常”等低信息词。
2. 复现步骤必须可复现，不得省略账号角色、页面路径和关键输入。
3. 实际结果需附带关键证据，接口问题至少保留请求和响应。
4. 影响范围要写明是否阻断主流程，是否影响权限或数据安全。
5. 若存在临时绕过方案，必须写入备注，便于测试和业务决策。

## 4. 缺陷填写示例

### 标题

`[BUG-20260306-001] [P0][审核流程] 管理员已分配专家后，专家待审核列表为空`

### 示例内容

- 提交日期：2026-03-06
- 提交人：QA
- 环境：SIT / Chrome 135 / 管理员+专家账号
- 模块：审核流程
- 优先级：High
- 前置条件：管理员已创建并提交一条成果，且已分配给专家账号
- 复现率：必现
- 状态：新建

#### 复现步骤

1. 管理员登录并进入审核分配页。
2. 将待审核成果分配给专家账号。
3. 专家登录并进入评审页。

#### 实际结果

专家待审核列表为空，无可审核记录。

#### 预期结果

专家应看到被分配的成果，并可以执行通过或驳回操作。

#### 影响范围

核心审核主流程阻断，成果无法完成评审闭环。
