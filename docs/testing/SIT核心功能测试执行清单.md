# ScienceManager SIT 核心功能测试执行清单

## 1. 目标

验证 ScienceManager 在 SIT 联调环境中的业务可用性、角色权限正确性、核心接口联通性，以及过程系统到管理端展示的主链路稳定性。

## 2. 角色与测试数据准备

### 2.1 角色账号

| 角色 | 用途 | 备注 |
|---|---|---|
| 普通用户 | 创建成果、提交审核、申请访问 | 需具备至少 1 个可关联项目 |
| 专家 | 审核通过、审核驳回 | 需可进入专家评审页 |
| 管理员 | 成果类型配置、审核分配、访问审批、过程成果查看 | 需可访问所有管理页 |
| 过程系统调用方 | 过程成果上传、幂等验证 | 需持有有效 API Key |

### 2.2 测试文件

| 文件类型 | 用途 |
|---|---|
| `pdf` | 合法附件上传 |
| `docx` | 合法附件上传 |
| `xlsx` | 合法附件上传 |
| `zip` | 合法附件上传 |
| 超大文件 | 文件大小边界验证 |
| 非法扩展名文件 | 上传拦截验证 |

### 2.3 基础数据

- 至少 1 个有效项目
- 至少 2 个成果类型
- 至少 1 套动态字段配置
- 至少 1 条受限成果数据

## 3. 执行步骤

### 步骤 1：环境准备

1. 确认前端、后端、Strapi、Keycloak、MySQL、Redis 均已启动且可访问。
2. 确认普通用户、专家、管理员账号均可登录。
3. 确认过程系统 API Key、`type_code`、幂等键测试策略可用。
4. 确认测试文件可上传，导出接口允许调用。
5. 记录本轮测试环境信息：

| Strapi 地址 |  |

| 数据库版本 |  |
| Redis 版本 |  |
| 构建版本/分支 |  |
| 执行日期 |  |

### 步骤 2：冒烟验证

1. 普通用户登录，确认首页、个人成果、检索页可访问。
2. 专家登录，确认评审页可访问。
3. 管理员登录，确认成果管理、审核分配、成果类型配置、过程成果页可访问。
4. 退出登录后刷新受保护页面，确认被重定向到登录页或无权限页。

### 步骤 3：成果类型与动态字段

1. 管理员创建一个新成果类型。
2. 为该类型配置文本、枚举、日期、附件等动态字段。
3. 普通用户进入成果创建页，选择新类型。
4. 验证字段渲染、默认值、必填校验、异常提示是否正确。

### 步骤 4：普通用户成果主流程

1. 新建成果并保存草稿。
2. 打开草稿进行编辑并提交审核。
3. 在“我的成果”中确认状态从草稿变为待审核。
4. 打开详情页，确认基本信息、动态字段、附件、可见范围一致。

### 步骤 5：管理员与专家审核流程

1. 管理员在待审核列表中找到该成果并分配专家。
2. 专家进入待审核列表，执行一次“通过”验证。
3. 再准备一条成果，执行一次“驳回”验证。
4. 普通用户查看驳回意见并修改后重新提交。
5. 管理员确认最终状态、列表筛选结果和详情页内容一致。

### 步骤 6：权限与访问申请

1. 普通用户直接访问 `/admin/*` 页面，确认被拒绝。
2. 普通用户对受限成果发起全文访问申请。
3. 管理员在访问申请页执行审批通过。
4. 普通用户再次访问对应成果全文，确认授权后可见。
5. 在审批前直接访问详情或下载接口，确认无越权读取。

### 步骤 7：过程系统联调主链路

1. 调用 `/api/v1/process-system/achievements/createWithFiles` 创建过程成果。
2. 使用相同幂等键与相同请求重放一次，确认命中幂等。
3. 使用相同幂等键但修改请求内容，确认返回冲突。
4. 管理员进入 `/admin/interim-results`，确认列表可见、详情可开、附件可下载或预览。
5. 详细接口断言参考 [过程系统API测试文档](/Users/junhaowu/Desktop/tongxiang_project/ScienceManager/achmanager-backend/docs/过程系统API测试文档.md)。

### 步骤 8：检索、导出与异常提示

1. 在成果列表按关键词、类型、状态进行筛选。
2. 验证分页、排序和空结果提示。
3. 管理员导出成果列表，确认文件下载成功且字段完整。
4. 制造必填缺失或权限不足场景，确认前后端错误提示明确。

### 步骤 9：收尾与回归

1. 回归主链路：登录、创建成果、提交审核、专家评审、过程成果展示。
2. 汇总缺陷并按严重级别分组。
3. 输出测试结论：通过、限制通过、不通过。

## 5. 出口标准

- P0、P1 缺陷已关闭或有明确业务批准的绕行方案。
- 核心主链路全部执行完成。
- 权限、审核、过程系统联调链路无阻断性问题。
- 测试结论和缺陷清单已同步到项目管理工具。
